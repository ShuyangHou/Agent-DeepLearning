## 2. 模块在课程中的位置

T07 已经把"误差顺着前向连接逆着回去"这件事铺开了。T08 要把那条"回去"的路径放大、拆开、再拼起来，让玩家明白：
反向传播之所以成立，是因为链式法则允许我们把 Loss 对远处参数的偏导，分解为沿途几段"局部影响"的乘积。

教学的关键认知：

> 不要直接做整条路径的微分。
> 先做"轻轻拨一下，看下一个量怎么变"的局部实验，
> 再把每一段实验的结果按顺序相乘，
> 就得到远处参数对 Loss 的总影响。

在课程整体结构里，T08 是反向传播的数学合法性证明，也是 T13–T18 CNN 反向传播、第五章 RNN BPTT 的共同基础。

---

## 3. 不讲什么

这个模块不涉及：

- 全部偏导的显式推导（让玩家算导数不是目标）
- 张量、Jacobian、向量化形式
- softmax / 交叉熵等具体损失函数的梯度
- 多分类、批维度
- 自动微分系统（autograd）的工程实现
- 反向传播的实现代码（那是 T07 已经讲过的视觉直觉部分）

一句话定位：

> T08 只解决"为什么可以这样回传"的合法性问题，把链条拆给玩家看。

---

## 4. 教学路径

## 玩家旅程

1. 把 T07 里那条"反向传播"的回传路径整条放大、单独拎出来。
2. 强调链条是有方向的：`x → z → h → ŷ → Loss`，反过来就是 `Loss → ŷ → h → z → w`。
3. 在每一段上做"轻轻拨一下"的局部影响实验，告诉玩家这就是"局部偏导"的直觉。
4. 把四段局部影响按顺序拼起来，乘出 dLoss/dw。
5. 把这套办法应用到多个隐藏神经元上，让玩家看到反向梯度会按连接强度自然分流。
6. 给玩家一个简单的概念测验：链式法则 = "沿途每段局部影响连续相乘"。

认知阶梯：

```text
整条路径太复杂
   ↓
只看一小段的局部变化
   ↓
把局部相乘
   ↓
得到远处参数的总影响
   ↓
复用到隐藏层的每个神经元
```

## 5. 计算图与最小例子

沿用 T07 的 MLP 骨架：

```text
x  →  隐藏层 3 个神经元  →  输出 ŷ
```

T08 把第一个隐藏神经元单独拎出来，展开成一条计算链：

```text
x
│  z = w · x + b
│  h = activation(z)
│  ŷ = v · h + c
│  Loss = (ŷ − y)^2
```

并强调：这条链上的每一段（`w→z`、`z→h`、`h→ŷ`、`ŷ→Loss`）都是一个简单的局部函数，导数直观可读。

## 6. 画面 1：把回传路径放大成一条链

### 页面目标

从 T07 的整张反向传播网络中，把"输入侧权重 `w` 一路影响到 `Loss`"的那条路径单独拎出来放大，让玩家先在视觉上接受："这是一条有方向的计算链"。

### 页面元素

- 上方：T07 风格的 MLP 缩略图（灰色淡出），上面有一条彩色高亮路径。
- 中央：放大后的链条 `w → z → h → ŷ → Loss`，五个节点四段边。
- 左下：节点说明

```text
w：要更新的权重
z = w · x + b：仿射前激活
h = activation(z)：神经元输出
ŷ = v · h + c：模型预测
Loss = (ŷ − y)^2：误差
```

- 右下：按钮 `开始拨动局部`。

### 交互规则

- 玩家点击 `开始拨动局部`，进入画面 2。
- 在这一画面玩家不需要算任何东西，只需要看清这条链的形状。

### 用户体验重点

第一次让玩家感受到："不是整张网络一起想，而是先沿着这一条链看清楚"。

## 7. 画面 2：在每一段上做"局部影响"实验

### 页面目标

让玩家亲手在链条的每一段上拨动前一个量，观察后一个量怎么变；把"局部偏导"的概念落到"轻轻拨一下"的直觉上。

### 页面元素

- 中央：链条 `w → z → h → ŷ → Loss`，四条边可点击。
- 每条边对应一张"局部影响卡"：
  - `∂z/∂w`：w 动一下，z 同向变。
  - `∂h/∂z`：z 动一下，看激活曲线决定 h 怎么变。
  - `∂ŷ/∂h`：h 动一下，输出权重决定 ŷ 怎么变。
  - `∂Loss/∂ŷ`：ŷ 动一下，误差线变长 / 变短，Loss 跟着改。
- 右侧：当前选中那一段的局部图示（小坐标轴 + 切线）。
- 底部反馈条：根据当前选中段给出一句"局部直觉"。

### 交互规则

- 玩家依次点击四条边，每点一条：
  - 对应的边变亮、节点出现微小的"上下 / 左右"动画。
  - 反馈条更新一句话（例如："w 轻轻变大，z 也跟着变大；这就是 ∂z/∂w。"）。
  - 该段的局部偏导卡片高亮一次。
- 四条边都被点过后，出现按钮 `把四段拼起来`，进入画面 3。

### 用户体验重点

让玩家形成 "导数 = 局部影响" 的直觉，而不是先看一堆 dx/dy 的公式。

## 8. 画面 3：把四段局部影响拼成一条乘法

### 页面目标

让玩家亲眼看到链式法则的核心结论：

```text
dLoss/dw = ∂Loss/∂ŷ · ∂ŷ/∂h · ∂h/∂z · ∂z/∂w
```

不是死背公式，而是"沿途每一段拿一份，按顺序乘起来"。

### 页面元素

- 中央：链条 `w → z → h → ŷ → Loss` 上方依次浮出四张局部偏导卡：

```text
[ ∂z/∂w ] [ ∂h/∂z ] [ ∂ŷ/∂h ] [ ∂Loss/∂ŷ ]
```

- 下方：链式法则拼接区，初始为空：

```text
dLoss/dw = [ ] × [ ] × [ ] × [ ]
```

- 右下角：按钮 `合成总影响`、`再做一次`、`下一画面`。

### 交互规则

- 玩家点击 `合成总影响`：
  - 四张卡按顺序飞入拼接区。
  - 卡片之间出现乘号。
  - 拼接区合成出一个总梯度卡 `dLoss/dw`。
  - 反馈条显示：

```text
链式法则就是把沿途每一段局部影响乘起来，得到远处参数对 Loss 的总影响。
```

- 玩家可点击 `再做一次` 重新演示。
- 玩家点击 `下一画面` 进入画面 4。

### 用户体验重点

把"链式法则"从一行公式还原成"四段局部影响连续相乘"的视觉动作。

## 9. 画面 4：隐藏层也能拿到梯度

### 页面目标

让玩家理解：在多神经元 MLP 中，输出层算出的梯度会自然分流给每一个隐藏神经元，每个 (w_i, b_i) 都能通过同样的链式相乘拿到自己的方向。

### 页面元素

- 左侧：输入 `x`。
- 中央：3 个隐藏神经元 `h1 / h2 / h3`。
- 右侧：输出 `ŷ`、`Loss`。
- 输出层到隐藏层的连接：粗细代表连接权重大小。
- 上方：紫色反向梯度信号从 `Loss → ŷ` 出发，到达输出层后分成三股流向 `h1 / h2 / h3`。
- 右侧梯度表（初始为空）：

```text
w1 / b1 ：待定
w2 / b2 ：待定
w3 / b3 ：待定
```

### 交互规则

- 玩家点击 `回传到隐藏层`：
  - 紫色梯度信号沿连接逆向流动到 `h1/h2/h3`，强度按连接权重显示。
  - 反馈条：

```text
输出层会把 Loss 的影响分给每个隐藏神经元；连接越关键，回传越强。
```

- 玩家依次点击 `h1 / h2 / h3` 每个神经元：
  - 高亮出对应的局部链条 `w_i / b_i → z_i → h_i → ŷ → Loss`。
  - 梯度表里 `w_i / b_i` 这一行填入方向箭头与建议（增大一点 / 减小一点）。
  - 反馈条：

```text
h_i 的参数仍然遵守同一条链式法则，只是把局部段拼上"输出层共享回传"。
```

- 三个神经元都看过后，出现按钮 `下一画面`。

### 用户体验重点

让玩家看到链式法则在多神经元情形下的"复用"：上一段是共享的，下一段各自不同；这是反向传播算法之所以高效的本质原因。

## 10. 画面 5：概念测验

### 页面目标

用一道概念题确认玩家是否掌握了"链式法则 = 沿途局部影响连续相乘"这个核心结论。

### 页面元素

- 题干：

> 关于链式法则在反向传播里的作用，下列说法最准确的是？

- 三个选项：
  - A. 链式法则会一次性给出每个参数对 Loss 的精确数值，无需关心中间量。
  - B. 链式法则把 Loss 对远处参数的偏导拆成沿途每一段局部偏导的乘积，分段算、再相乘。
  - C. 链式法则只在隐藏层有效，输出层和 Loss 之间不需要它。
- 反馈区。

### 交互规则

- 选 B：高亮为正确，反馈："对，这就是反向传播之所以成立的原因。"
- 选 A / C：高亮为错误，反馈："想一下：链式法则是把每一段局部影响乘起来；它不会跳过中间量，也不只对某一层有效。"
- 答对后，整个 T08 标记为完成。

### 用户体验重点

让玩家用自己的语言把"链式法则"重述一遍，并锁定核心结论。
