## 2. 模块在课程中的位置

T07/T08 解决了"每一步参数怎么算出方向"这件事——给定一批样本，反向传播会按链式法则给出梯度。T09 要回答的是上游问题：

> 既然每一步都需要一份样本，那这份样本应该用多少？多久算一步？什么时候算"训练了一轮"？

教学的关键句：

> 一次"前向 + 反向 + 更新" = 一步（step / iteration）；
> 这一步用的那份样本就是一个 **batch**；
> 把整个训练集按 batch 切好、依次走一遍 = 一个 **epoch**；
> 多个 epoch 串起来 = 训练循环。

在课程整体结构里，T09 是 T10（学习率）和 T11（欠拟合/过拟合）的前置：学习率作用在"每一步"上，欠拟合/过拟合作用在"多少个 epoch"的尺度上，两者都建立在 T09 的时间轴上。

## 3. 不讲什么

这个模块不涉及：

- 学习率的具体设置（留给 T10）
- 优化器（SGD / Momentum / Adam）的差异
- DataLoader 的工程实现、shuffle 的随机数细节
- batch normalization、gradient accumulation 等进阶技巧
- 任何具体的损失函数推导（在 T02 已经讲过）

一句话定位：

> T09 只解决"一次训练循环到底由哪些时间尺度组成"的概念问题，把 step / batch / epoch 这三个词钉死在玩家脑子里。

## 4. 教学路径

## 玩家旅程

1. 先看到完整数据集，理解"全量梯度"在直觉上的样子（一次算完所有 N 个样本）。
2. 抛出问题：N 很大的时候，每一步都跑完全部样本既慢又不灵活。
3. 引入 mini-batch：把数据集打散，切成大小为 B 的小份；每一份做一次前向+反向+更新，就是一步。
4. 把"步"串起来：N/B 步刚好走完整个数据集 → 这就是一个 epoch。
5. 把 epoch 串起来：每个 epoch 重新洗牌，loss 一轮一轮往下走。
6. 用一个对比实验说明 batch size 的取舍：B=1 噪声大但灵活、B=N 平滑但慢、B=中等是常用折中。
7. 小测验确认玩家把 step / batch / epoch 三者区分清楚。

认知阶梯：

```text
整个数据集一次算完？
   ↓
太大太慢，先切成小份
   ↓
每个小份走一步 → step
   ↓
所有小份串成一轮 → epoch
   ↓
多个 epoch 把 loss 推下去
```

## 5. 数据集与最小例子

用一份示意数据集：

```text
训练集大小 N = 24
batch 大小 B = 4
一个 epoch 的步数 = N / B = 6 步
训练 3 个 epoch = 18 步
```

视觉上把这 24 个样本画成一个 4×6 的方阵；每次取 B=4 个样本（颜色高亮）算一步。这样玩家能在屏幕上数清楚：步、batch、epoch 各自对应多少格子。

对比实验中用到的三个 batch size：

- B = 1（最噪声、最灵活）
- B = 4（折中）
- B = 24（全量，最平滑）

## 6. 画面 1：完整数据集与"全量"的代价

### 页面目标

让玩家先在视觉上看到"训练集是一堆样本的集合"，并理解：如果每一步都吃掉全部 N 个样本算一次梯度，逻辑上没问题，但代价是"每动一下都要算所有人"。

### 页面元素

- 中央：一个 4×6 的样本方阵（24 个圆点），代表训练集。
- 上方：标签 `训练集 N = 24`。
- 右侧面板：
  - 一行小公式：`全量梯度 = mean over all N samples`。
  - 一个按钮 `算一次全量梯度`。
  - 一行说明：`这种方式叫 full-batch / batch gradient descent`。
- 底部反馈条。

### 交互规则

- 玩家点击 `算一次全量梯度`：
  - 所有 24 个样本同时高亮 → 汇聚到一支大箭头（表示这一步的梯度）。
  - 反馈条提示：`这一步用到了全部 24 个样本；规模大时这样做太慢`。
- 出现 `下一画面 →`，进入画面 2。

### 用户体验重点

先让玩家承认"全量"的直觉，再引出后续"为什么要切批"。

---

## 7. 画面 2：把数据集切成 batch，走一步

### 页面目标

把 mini-batch 的概念引入：洗牌 → 切成大小为 B 的若干份 → 每份做一次"前向 + 反向 + 更新" → 这就是一步。

### 页面元素

- 中央：4×6 数据方阵，下方有一条传送带样的 "batch 队列"，长度 6 格，每格代表一个 batch（B=4）。
- 右侧面板：
  - 三个控件：
    - `Batch 大小 B`：可在 {1, 2, 3, 4, 6, 8, 12, 24} 中切换。
    - 按钮 `洗牌`：随机打散方阵编号。
    - 按钮 `走一步`：取出队列里的下一个 batch，跑一次 step。
  - 一块"step 状态"区，显示：
    - 当前 step 编号
    - 这一步用到的样本编号
    - 这一步的梯度方向（用一支小箭头示意）
- 底部反馈条。

### 交互规则

- 玩家调整 B：队列长度立即更新为 `N / B` 格，方阵的高亮分组也跟着更新。
- 玩家点击 `洗牌`：方阵里的编号被随机重排，batch 队列重置。
- 玩家点击 `走一步`：
  - 队列最左边的 batch 弹出，对应的 B 个样本在方阵中高亮一下。
  - "step 状态" 区刷新：step 编号 +1，方向箭头摆一下。
  - 反馈条提示：`第 k 步：吃了一个大小为 B 的 batch`。
- 当队列消耗到 0 时：弹出按钮 `把这一轮串起来 →`，进入画面 3。

### 用户体验重点

让玩家亲手"走步"，把"一步 = 一个 batch 的前向+反向+更新"这一对应关系刻进肌肉记忆。

## 8. 画面 3：N/B 步 = 一个 epoch

### 页面目标

把"步"串起来，告诉玩家：把整份洗好的数据集按 batch 走一遍，就是一个 epoch；epoch 是"数据集层面"的时间单位，step 是"参数更新层面"的时间单位。

### 页面元素

- 中央：4×6 数据方阵 + 下方 6 格 batch 队列（沿用画面 2，B 固定为 4）。
- 上方有一条"epoch 时间轴"，6 段刻度，每段对应一步。
- 右侧面板：
  - 按钮 `自动跑完一个 epoch`。
  - 进度文字：`已经走了 k / 6 步`。
  - 概念卡：
    - `1 step = 1 个 batch`
    - `1 epoch = N / B 个 step`
    - `这里 N=24, B=4, 1 epoch = 6 步`
- 底部反馈条。

### 交互规则

- 玩家点击 `自动跑完一个 epoch`：
  - 队列被逐个吃掉，时间轴格子按顺序填满。
  - 每填一格，方阵中对应的 B 个样本高亮一下。
  - 填满时反馈条：`6 步走完了整个数据集——这就是 1 个 epoch`。
- 出现按钮 `多跑几个 epoch →`，进入画面 4。

### 用户体验重点

让玩家彻底分清 step 与 epoch：step 是"单步更新"，epoch 是"完整遍历一遍数据"。

---

## 9. 画面 4：多个 epoch 串起来 + batch size 的取舍

### 页面目标

把训练循环放进真实尺度：连续跑 E 个 epoch，loss 一轮一轮下降；并对比不同 batch size 下 loss 曲线的"噪声 / 平滑"差别，让玩家形成判断。

### 页面元素

- 上方：epoch 时间轴扩展为 3 个 epoch，每个 epoch 6 段，共 18 段；颜色按 epoch 分色。
- 中央：一张折线图，x 轴是 step，y 轴是 loss。
- 右侧面板：
  - `Batch 大小` 三选一切换：`B=1` / `B=4` / `B=24`。
  - 按钮 `跑 3 个 epoch`。
  - 概念卡：
    - `B 小：每步噪声大，但灵活、能跳出局部小坑`
    - `B 大：每步平滑、方向准，但每步算得慢`
    - `B 中：折中，最常用`
- 底部反馈条。

### 交互规则

- 玩家选择 B：折线图样式（噪声强度、每个 epoch 内的步数）随之变化。
- 玩家点击 `跑 3 个 epoch`：
  - 时间轴按顺序填色，每填一段，对应的 loss 点在折线图上落下。
  - 跑完时反馈条根据 B 给出不同说法：
    - B=1：`非常颠簸，但能感觉到方向`
    - B=4：`折中：曲线相对平稳，又不像全量那样僵`
    - B=24：`平滑但每个 epoch 只能更新 1 次，速度慢`
- 出现 `进入小测验 →`，进入画面 5。

### 用户体验重点

让玩家用眼睛比较 batch size 对训练动态的影响，建立"B 是个权衡变量"的直觉。

## 10. 画面 5：概念测验

### 页面目标

用一道概念题确认玩家是否掌握了"step / batch / epoch 三者关系"这个核心结论。

### 页面元素

- 题干：

> 训练集有 N=1000 个样本，设 batch size B=100。下列说法正确的是？

- 三个选项：
  - A. 一个 epoch 等于 1 步，因为每个 epoch 都只更新一次参数。
  - B. 一个 epoch 等于 10 步，每一步用 100 个样本做一次前向+反向+更新；跑 5 个 epoch 一共更新 50 次。
  - C. batch size 越大越好，因为方向更准。
- 反馈区。

### 交互规则

- 选 B：高亮为正确，反馈：`对了：1 epoch = N/B 步；epoch 是"过完一遍数据"的尺度，step 是"更新一次参数"的尺度。`
- 选 A：反馈：`想想画面 3：epoch 是按 batch 走完整遍数据，不是只走一步。`
- 选 C：反馈：`画面 4 已经展示过：B 大方向稳但慢、B 小灵活但抖；B 是个折中。`
- 答对后整个 T09 标记完成。

### 用户体验重点

让玩家自己说出 step / batch / epoch 的关系，把概念锁死。
