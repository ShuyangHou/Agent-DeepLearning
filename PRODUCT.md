# Product

## Register

product

## Users
本科 / 研究生阶段的深度学习课程学生与助教；目标是在浏览器里"亲手把一个抽象训练概念跑通一遍"。使用场景多为课堂演示、实验课、自学预习。屏幕从教室投影到笔记本到平板都要清晰。

## Product Purpose
用一组互动模块（一关一概念）把"反向传播 / 链式法则 / 学习率 ..."这些训练机制变成可点、可拖、可观察的小实验：从直觉到公式到曲线，让学生用十分钟左右走完一个核心概念，并留下一条 Loss 曲线、一组参数变化、一段 manim 视频作为记忆抓手。

## Brand Personality
克制、严肃、可信。三词速写：克制专业、教学优先、有节奏。语气接近一本好讲义：先讲清楚要做什么，再让你动手，最后一句话收束概念。不要营销腔、不要装饰性插画、不要"AI 风" cream/sand 配色或大渐变 hero。

## Anti-references
- 任何 SaaS landing page 风格的英雄大字 + 渐变背景 + 营销 CTA。
- 大量装饰性 emoji、表情包、彩虹 badge。
- "AI 教程页" 常见的奶油色 + 大圆角 + 软阴影 ghost card 堆叠。
- 把概念塞进无差别 3 列卡片（icon + 标题 + 段落）的 SaaS 落地页结构。
- 任何 `border-left: 6px solid` 侧条色块、文字渐变、玻璃拟态。

## Design Principles
1. **概念优先**：每一屏只服务一个概念目标，多余装饰先砍。
2. **公式可点**：把公式 / 网络 / 曲线 / 滑杆放在同一块画布上互相联动，不要让玩家在文字里推理。
3. **颜色即语义**：蓝 = 前向 / 当前状态，紫 = 梯度，绿 = 更新 / 下降，红 = Loss / 上升，橙 = 学习率或被操作的参数。一旦定下不要在装饰里乱用。
4. **反馈条不撒谎**：底部反馈条说"再点一次"就真的还没解锁，说"很好"就真的过了门槛。
5. **进度即承诺**：顶部进度条是这一关的导航，不是装饰；解锁条件清晰，已完成有视觉确认。

## Accessibility & Inclusion
- 目标 WCAG AA：正文 ≥ 4.5:1，大字 / 重点色 ≥ 3:1。
- 所有动画提供 `prefers-reduced-motion` 退化（必要时改为瞬时切换）。
- 颜色不是唯一信息载体：状态条同时用形状 / 文字 / 图标说明（不只是变色）。
- 中文为主，字体栈优先 `PingFang SC` / `Hiragino Sans GB`，避免被宋体或楷体替换。
